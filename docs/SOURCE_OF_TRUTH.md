## 10. Audit checklist (apply when reviewing any page)

When auditing a page, ask of every claim:
1. Is this from this doc?
2. If not, is there a real source? Add to this doc first.
3. If no source, REMOVE the claim (don't soften it, don't make it generic, REMOVE).

When auditing a photo/image:
1. Is it in `/real-photos/`, `/test-images/`, or `/catalog-images/`?
2. If it's a person — do we have permission and a real name?
3. If it's a stock-feeling photo, kill it.

When auditing copy:
1. Read it as a Wausau homeowner. Does it sound like marketing fluff or a real neighbor talking?
2. If it says "premier" "industry-leading" "best in class" — rewrite.
3. If it makes up a number (4,200 reviews, 25 years, 100+ employees) — verify or remove.
